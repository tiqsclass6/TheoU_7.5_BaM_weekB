## 🛠️ **Troubleshooting Guide**

### ❗ **Git “command not found”**

Git is not installed or not in your PATH.

**Fix:**

```bash
sudo apt update
sudo apt install git -y
````

### ❗ **Username/Email not set**

You see:

```plaintext
Please tell me who you are.
```

**Fix:**

```bash
git config --global user.name "Your Name"
git config --global user.email "your@email.com"
```

### ❗ **Permission denied (publickey) when pushing**

SSH key missing or not added to GitHub.

**Fix:**

```bash
ssh-keygen -t ed25519 -C "you@example.com"
cat ~/.ssh/id_ed25519.pub
```

Add the output to:
**GitHub → Settings → SSH & GPG Keys → New SSH Key**

### ❗ **Merge conflict detected**

Git halts your merge until conflicts are resolved.

**Fix Steps:**

1. Open the file with conflict markers
2. Resolve the conflicting sections
3. Stage and commit the result

```bash
git add .
git commit
```

### ❗ **Accidentally committed the wrong file**

Undo the commit but **keep your work**:

```bash
git reset --soft HEAD~1
```

Undo the commit and **delete changes**:

```bash
git reset --hard HEAD~1
```

---

### ❗ **Want to unstage files but keep your work**

```bash
git reset HEAD
```
